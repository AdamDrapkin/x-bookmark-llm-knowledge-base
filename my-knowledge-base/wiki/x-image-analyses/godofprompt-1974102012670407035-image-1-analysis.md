### Key Framework Components

**D - Define Multiple Perspectives**
- Use multiple expert viewpoints instead of single role
- Example: "You are three experts: behavioral psychologist, copywriter, and data analyst"
- Creates richer, more nuanced responses

**E - Establish Success Metrics**
- Define specific, measurable outcomes upfront
- Instead of "make it good" → "optimize for 40% open rate, 12% CTR"
- Gives AI clear targets to hit

**P - Provide Context Layers**
- Stack multiple context elements: business type, budget, audience, past performance
- More detail = more targeted output

**T - Task Breakdown**
- Break requests into sequential steps
- Example: "Step 1: Identify pain points. Step 2: Create hook..."
- Creates logical flow in output

**H - Human Feedback Loop**
- Build in self-critique
- Example: "Rate your response 1-10 on clarity, persuasion, actionability"
- Iterative improvement built into single prompt

### Summary (TD;LR)
1. Single-role prompts = generic outputs - Always use multiple perspectives
2. No metrics = no optimization - Specify exact success criteria
3. Context is king - More background = better relevance
4. Break down tasks - Step-by-step prevents confusion
5. Force self-evaluation - AI critiquing itself improves quality dramatically
