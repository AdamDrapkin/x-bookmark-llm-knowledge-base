## Summary

Kapso CLI represents a bridge between [[AI agents]] and [[WhatsApp]]'s massive user base, enabling developers to give their autonomous agents their own phone numbers for communication. The tool is positioned as a simple solution for agent-to-user communication — by attaching a WhatsApp number to an agent, developers can leverage WhatsApp's messaging infrastructure without building custom integrations.

The setup process is intentionally minimal: after installing the package globally via npm, a single `kapso setup` command initializes the connection. This two-step workflow suggests the tool abstracts away the complexity of WhatsApp Business API authentication and number verification that typically plagues such integrations.

The launch targets developers building [[agentic applications]] who want their AI assistants to communicate with users on WhatsApp — whether for customer support, personal assistants, or automated services. WhatsApp's end-to-end encryption and widespread adoption (particularly in markets outside the US) make it an attractive channel for agent-user interactions.

## Key Takeaways

- Kapso CLI provides [[WhatsApp Business API]] integration for AI agents through a simplified CLI interface
- Two-command setup: `npm install -g @kapso/cli` followed by `kapso setup`
- The tool assigns dedicated WhatsApp numbers to agents, enabling them to send and receive messages
- Targets [[agentic AI]] developers looking to deploy agents on WhatsApp as a communication channel
- Published under the npm package `@kapso/cli`
