## Summary

This tweet references [[andrej-karpathy]]'s "second brain" system for AI-powered personal knowledge management. The architecture is elegantly simple: three folders that form a pipeline where raw information flows in one end and structured, queryable knowledge comes out the other.

The system uses a single [[claude.md]] file as the schema/blueprint that tells the [[ai-agent]] how to organize information. The three-folder structure creates a deliberate separation between input (raw dumps), processing (AI-organized wiki), and retrieval (query outputs). This is a practical implementation of how [[large-language-models]] can serve as the organizing layer between human information capture and usable knowledge synthesis.

## Key Takeaways

- Three-folder architecture provides a clear mental model: dump → organize → query
- [[claude.md]] acts as the schema/contract that defines how AI should structure and connect knowledge
- The system is self-referential—AI organizes information for itself to use
- Emphasizes the 80/20 principle: minimal structure, maximum utility
- Automation happens at the "organize" step, reducing manual curation overhead
- The [[outputs/]] folder represents the queryable interface—AI answers questions from the organized wiki
