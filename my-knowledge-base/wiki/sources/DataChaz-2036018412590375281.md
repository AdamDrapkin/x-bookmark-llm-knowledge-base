## Summary

Developer @louisvarge has created `claude-peers`, an open-source framework that enables multiple [[Claude]] sessions to communicate and collaborate locally, effectively transforming a single AI instance into a distributed development team. The system leverages a local [[SQLite]] daemon as the backbone for peer synchronization, while [[MCP]] (Model Context Protocol) servers handle instant message pushing between agents. This architecture allows different AI agents to maintain shared context, delegate tasks, and auto-summarize their collective work in real-time. The approach is particularly notable for its emphasis on local execution—keeping all agent communication private and under user control, without relying on external cloud services for inter-agent messaging.

## Key Takeaways

- Enables multiple [[Claude]] sessions to chat and collaborate locally as a dev team
- Uses [[SQLite]] daemon for reliable peer-to-peer state synchronization
- [[MCP]] servers provide real-time message delivery between agents
- Auto-summarizes collaborative sessions for context continuity
- Fully local execution preserves privacy and reduces latency
- Open-source project by @louisvarge
