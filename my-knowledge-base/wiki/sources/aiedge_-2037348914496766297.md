## Summary

This tweet shares a comprehensive cheatsheet of every Claude Code command, organized into 12 functional categories covering 93 total commands. The reference spans from essential project management commands like `/init` and `/memory` for working with the CLAUDE.md file, through information utilities like `/usage` and `/cost` for tracking token consumption, to advanced features like `/agents` for creating sub-agents and `/sandbox` for isolated execution environments. The document serves as a complete quick-reference for developers using Anthropic's CLI tool, covering everything from basic session management (`/resume`, `/fork`, `/rename`) to model control (`/model`, `/fast`, `/plan`) and environment customization (`/theme`, `/keybindings`, `/terminal-setup`). The reference was published March 2026 and appears to be the most comprehensive public documentation of Claude Code's command structure.

## Key Takeaways

- **Project Management** commands (1-12) handle session control, context management, and file operations including `/compact` for freeing context space, `/diff` for viewing changes interactively, and `/export` for saving conversations
- **Information & Status** commands (13-21) provide visibility into usage with `/usage`, `/cost`, and `/stats`, plus diagnostic tools like `/doctor` for environment checks and `/debug` for troubleshooting
- **Mode & Model Control** (22-27) enables switching between Opus, Sonnet, and Haiku models, toggling Fast Mode for 2.5x speed on Opus 4.6, and accessing read-only Plan Mode
- **Feature Management** (28-35) includes advanced capabilities like `/hooks` for lifecycle automation, `/agents` for parallel sub-agent execution, and `/rewind` to revert conversation state
- **Environment Settings** (36-40) and beyond cover terminal customization including keybindings, themes, and status line configuration
- The **Integrations & External Tools** category expands Claude Code's reach beyond the CLI into broader development workflows
