## Summary

This cheatsheet addresses a fundamental challenge in long AI workflows: **context window overload**. When conversations grow lengthy, critical instructions get buried, the model wastes tokens on irrelevant content, and responses become inconsistent or misaligned with specialized procedures.

[[Claude Skills]] solve this through **externalization and modularization** of procedural knowledge. Rather than cramming everything into the context window, Skills allow Claude to load only metadata (name, description) upfront, then pull in task-specific instructions and code only when relevant. This keeps the context lightweight and focused, freeing tokens for actual reasoning and user data.

The architecture leverages **dynamic loading** — Claude automatically scans and activates relevant Skills — combined with **progressive disclosure**, which loads metadata first before fetching full content. Skills are designed to be **composable** (multiple can stack and coordinate), **portable** (same structure across Claude Apps, Code, and API), and **executable** (can run code in a sandbox for deterministic results).

Each Skill follows a standardized folder structure containing `skill.md` (YAML frontmatter + Markdown instructions), a `Resources/` directory for reference docs and templates, and an optional `Scripts/` directory for executable code. The required YAML fields include Name, Description, Version, and Dependencies.

## Key Takeaways

- **Context window overload** is the core problem: long workflows suffer from buried instructions, token waste, and inconsistent outputs
- **Skills externalize procedural knowledge** rather than storing everything in context, enabling modular, reusable expertise
- **Dynamic loading** means Claude automatically activates relevant Skills without manual intervention
- **Progressive disclosure** optimizes token usage by loading metadata first, full content only when needed
- **Standardized folder structure** ensures portability: `skill.md` + `Resources/` + optional `Scripts/`
- **YAML frontmatter** must include: Name, Description, Version, and Dependencies
- Skills complement [[MCP]] (external tools/data), [[Projects]] (workspace context), and [[Custom Instructions]] (global persona) rather than replacing them
- Built-in Anthropic Skills cover Enhanced Excel, PowerPoint generation, and Word/PDF documents
