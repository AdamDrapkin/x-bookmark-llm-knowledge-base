# 724 Office - Self-Evolving AI Agent System

**Author:** @aigleeson

## Summary

724 Office represents a significant departure from the typical AI agent development approach. Rather than relying on heavy frameworks like [[langchain]], [[llamaindex]], or [[crewai]], this project achieves sophisticated autonomous agent behavior using approximately 3,500 lines of pure [[python]] combined with only three lightweight dependencies: [[croniter]] for scheduling, [[lancedb]] for vector retrieval, and [[websocket-client]] for real-time communication.

The system's most remarkable capability is **runtime tool creation** — the agent can autonomously write, save, and load new Python tools while operating. This transforms it from a static tool-execution system into a genuinely self-extending agent that can adapt its own capabilities based on task requirements. The self-repair mechanism runs daily self-checks, analyzes error logs, performs session health diagnostics, and automatically notifies operators when failures occur.

The three-layer memory architecture distinguishes between session-level context, LLM-compressed long-term storage, and LanceDB-powered vector retrieval — enabling both immediate responsiveness and persistent knowledge across sessions. The multi-tenant router uses [[docker]] containers with auto-provisioning, assigning each user their own isolated environment that can be health-checked and restarted independently. Combined with an OpenAI-compatible function calling interface supporting up to 20 automatic retry iterations, the system prioritizes reliability and fault tolerance in production 24/7 deployments.

## Key Takeaways

- Built with ~3,500 lines of pure Python using only the standard library plus three packages (croniter, LanceDB, websocket-client) — deliberately avoiding heavyweight AI frameworks
- Runtime tool creation enables the agent to write, save, and load new Python tools autonomously during operation, making it genuinely self-evolving rather than just tool-calling
- Three-layer memory system combining session history, LLM-compressed long-term memory, and LanceDB vector retrieval for comprehensive information management
- Self-repair capabilities include daily self-checks, error log analysis, session health diagnostics, and automatic failure notifications
- MCP/Plugin System supports external MCP servers via JSON-RPC (both stdio and HTTP modes) with hot-reload functionality requiring no restart
- Multi-tenant Docker architecture auto-provisions isolated containers per user with independent health-checking
- OpenAI-compatible function calling with automatic retry up to 20 iterations per conversation for robust tool execution
- Built solo with AI co-development tools in under 3 months, now running production 24/7
