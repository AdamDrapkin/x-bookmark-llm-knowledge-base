## Summary

HERETIC (v1.0.0) is an open-source tool published on GitHub that implements a targeted ablation technique to remove safety alignment from [[large language models]]. The tool works by identifying and neutralizing specific neural network components—particularly the `attn.o_proj` (attention output projection) and `mlp.down_proj` (MLP down projection) matrices—that encode refusal behaviors learned during [[RLHF]] (Reinforcement Learning from Human Feedback) or [[Constitutional AI]] training.

The process operates by comparing model responses to "harmless" versus "harmful" prompt datasets (using [[mlabonne/harmless_alpaca]] and [[mlabonne/harmful_behaviors]]). It measures first-token probability distributions across both datasets to identify which components trigger refusal behaviors. Once identified, these components are ablated (disabled), effectively removing the model's learned safety responses without requiring full retraining.

Running on an [[NVIDIA A100]] 80GB GPU with the [[openai/gpt-oss-20b]] model, HERETIC demonstrated reducing refusal rates from 97/100 to dramatically lower levels, completing the entire process in approximately 45 minutes. The tool optimizes batch processing up to 128 samples simultaneously, achieving throughput of 874 tokens/second during evaluation phases.

## Key Takeaways

- HERETIC uses **targeted neural network ablation** rather than retraining, making it computationally efficient
- The technique targets specific [[Transformer]] components (`attn.o_proj` and `mlp.down_proj`) across all 24 layers
- Safety removal requires only ~45 minutes on commodity hardware like the A100
- The tool achieves near-complete removal of refusal behaviors (97/100 initial refusals reduced significantly)
- It's an open-source project available at [[github.com/p-e-w/heretic]]
- Batch optimization allows processing 128 prompts in parallel for efficiency

## Technical Context

The HERETIC approach represents a significant advancement in [[model alignment removal]] techniques. Unlike earlier methods that required fine-tuning or full retraining, targeted ablation works by:

1. **Identifying refusal-inducing components**: By comparing probability distributions over "harmless" versus "harmful" prompts, the tool pinpoints which attention and MLP layers most strongly correlate with refusal tokens
2. **Selective component removal**: Rather than destroying the entire model, only specific weight matrices are nullified
3. **Preserving general capabilities**: Because ablation targets only safety-specific pathways, general language capabilities remain largely intact

This approach raises significant concerns about the accessibility of [[uncensored AI]] models. What previously required sophisticated ML expertise and extensive compute resources can now be accomplished in under an hour with a single command, potentially enabling misuse of formerly safety-aligned models.
