What is Claude Code?

An agent you run from the command line. You talk to it in natural language. It controls your computer.

Say "make a file on my desktop that says Hello, Greg" and it does it.

That's not the interesting part.

The interesting part: the more information you give it, the more complex things it can do.

But here's the problem most people hit:

They have a long conversation with an AI agent about a project. Then they start a new session. And they have to explain everything again.

—

What is Obsidian?

An interface that sits on top of a collection of markdown files.

What makes it different from a regular folder: Obsidian lets you connect relationships between files.

Write a daily note → link it to a project file → link that to a person file → over time, you get a web of interrelated ideas.

There's a graph visualization that shows every file as a node, connected to every other file it references.

A folder can't do that.

—

Why they work together

Claude Code can read files and create files.

Obsidian stores everything as markdown files.

So when you point Claude Code at your Obsidian vault, you're giving the agent access to your entire knowledge system: projects, preferences, reflections, notes.

You don't have to explain things over and over.

You write a project description once. Save it as a file. Reference it any time you start a new session.

Vin's line: "The whole game is feeding the beast good context."

—

The setup: Claude Code commands inside Obsidian

Vin created custom slash commands that run Claude Code directly from inside Obsidian.

How it works:

You write a system prompt as a markdown file in your vault.

That file becomes a command.

When you run it, Claude Code reads the prompt, scans the relevant files, and executes.

Examples Vin demonstrated live:

→ /reflect — scans your daily note and asks targeted reflection questions based on what you wrote

→ /extract — pulls key ideas from a daily note and creates standalone files for each one, with backlinks

→ /graduate — takes a recurring idea across multiple daily notes and promotes it into its own working document

These aren't generic AI prompts. They're contextual — because the agent is reading your notes, your patterns, your vault.

—

The daily workflow

Vin writes daily notes in Obsidian. Reflections, observations, ideas.

Then he runs commands against those notes.

The agent scans what he wrote, cross-references his vault, and surfaces patterns he wouldn't have noticed on his own.

He described it like having a parallel agent that watches your notes and gives you ideas on how to improve your workflow and your life.

And it doesn't just suggest it builds. New files, new connections, new commands.

—

Why markdown files matter

Vin made this point and it stuck:

"A file is essentially a perfect memory."

Human memory is unreliable. Studies show what we remember is often different from reality.

A markdown file is the exact thing you wrote at that exact moment. No decay. No bias beyond what you wrote.

And in a world where LLMs run on text, markdown files are the oxygen.

People think tokens are what matter. But the files — the memories — are what give the agent something meaningful to work with.

—

What's hard about this

It takes real time to set up. Obsidian is a blank canvas. There's no guided onboarding that says "write your preferences here."

You have to build your own system.

But Vin's framing: even when working with other humans, you have to find a way to explain things to them. The difference with agents — you only have to explain once. Write it into a file and it's there forever.

Action checklist:

Download Obsidian (free, open source)

Create your vault

Build your first project description file

Install Claude Code

Create one custom slash command using a system prompt file

Run it against your vault and see what comes back
